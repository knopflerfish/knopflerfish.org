Changes v0.6.0 2005-06-21

General Added copyright to all files.
Launch Config Added support for changing start level
Launch Config Removed unused buttons
Launch Config Fixed bug if two or more bundles had the same name only one was shown.
Launch Config Selected bundles are removed from available tree
Launch Config Multiselect allowed when adding/removing bundles
Launch Config Added possibility to set if bundle shall be started or installed. By default bundles with activators are started and those without installed.
Launch config Problems with missing packages are shown
Launch config Added possibilty to add exernal bundle
Project Wizard Bug fix, if no OSGi environment exists the wizard tried to read environment with name "" which resulted in that some sub nodes Bundles, Libraries and Properties where created.
Project Wizard, Preferences An classpath container is added to bundle projects classpaths. The container points either to the default KF installation or a speific one. The libraries in the container is specified in preference page for the specific KF installation. By default all bundles which have API in their Bundle-Category are added to the container.
Manifest Editor Added GUI for attribute Bundle-Category, Bundle-UpdateLocation
